# HEARTBEAT.md - 智能心跳检查清单

## 心跳模式说明

本 Agent 采用**智能双频心跳**机制：

| 模式 | 频率 | 触发条件 | 主要任务 |
|------|------|----------|----------|
| **高频巡查** | 每 2 分钟 | 有待处理任务或活跃任务时 | 检查任务进度、Agent状态、防止卡死 |
| **低频巡检** | 每 1 小时 | 无任务时 | 系统健康检查、上下文压缩优化 |

---

## 高频巡查清单（每 2 分钟）

执行条件：`MEMORY.md` 中有待审核任务 或 检测到活跃任务

### 1. 巡查其他 Agent 工作状态
- [ ] 检查 researcher、office、coder 的工作区文件修改时间
- [ ] 查看各 Agent 的 `HEARTBEAT.md` 是否有异常标记（如"需要帮助"）
- [ ] 若发现某个 Agent 超过 **10 分钟无输出**，主动介入询问：
  > "任务进展如何？是否需要协调？"

### 2. 检查任务状态
- [ ] 查看 `MEMORY.md` 中记录的待审核任务
- [ ] 确认是否有超时未审核的情况（超过 30 分钟）
- [ ] 检查是否有用户新消息需要响应

### 3. 系统健康检查
- [ ] 确认 `tavily` 工具可用
- [ ] 确认所有 Agent 技能依赖基本正常

### 4. 卡死检测与处理
- [ ] 检测各 Agent 是否响应超时（超过 30 秒）
- [ ] 如发现卡死，标记异常并准备重启会话

---

## 低频巡检清单（每 1 小时）

执行条件：无待处理任务

### 1. 系统健康检查
- [ ] 检查 Gateway 运行状态
- [ ] 确认所有 Agent 会话正常
- [ ] 检查磁盘空间使用情况

### 2. 上下文压缩优化
- [ ] 检查 manager 自身上下文长度
- [ ] 如上下文超过阈值（80%），触发压缩
- [ ] 记录压缩日志到 `MEMORY.md`

### 3. 资源清理
- [ ] 清理过期临时文件（超过 24 小时）
- [ ] 检查日志文件大小

### 4. 状态汇报
- [ ] 汇总所有 Agent 状态
- [ ] 记录系统健康状态到日志

---

## 任务状态标记

在 `MEMORY.md` 中使用以下标记：

```markdown
## 当前活跃任务
- [ ] 任务ID: xxx | Agent: researcher | 状态: 进行中 | 开始时间: 2026-03-15 14:00

## 待审核任务
- [ ] 任务ID: xxx | Agent: coder | 提交时间: 2026-03-15 14:30
```

---

## 异常处理流程

### Agent 无响应
1. 发送询问消息到对应 Agent
2. 等待 1 分钟响应
3. 如无响应，标记为"可能卡死"
4. 向用户报告情况

### 任务超时
1. 检查任务实际进度
2. 如确实停滞，协调资源或重新分配
3. 记录超时原因

### 系统异常
1. 记录异常详情
2. 尝试自动恢复
3. 如无法恢复，向用户报告

---

## 配置信息

- **高频 Cron Job**: `manager-heartbeat-high` (每 2 分钟)
- **低频 Cron Job**: `manager-heartbeat-low` (每 1 小时)
- **检查阈值**: 10 分钟无输出视为异常
- **响应超时**: 30 秒
