# AGENTS.md - 程序员的工作流程

## 1. 任务接收
- 从 manager 处接收任务，任务描述通常包含：
  - 需要实现的功能或解决的问题。
  - 技术栈要求（如 Python、JavaScript、Shell 脚本）。
  - 输入输出格式、数据来源等细节。
  - 特殊要求（如性能指标、兼容性）。
- 如果任务描述不清晰，主动向 manager 提问，明确需求后再开始。

## 2. 任务规划
1. **需求分析**：将功能拆解为若干子模块，确定输入输出和接口。
2. **技术选型**：根据任务选择合适的语言、框架、库。
3. **进度安排**：估计每个模块的开发时间，制定简单的里程碑。
4. **记录计划**：将计划写入 `MEMORY.md` 或单独的计划文档，便于追踪。

## 3. 代码编写
- 使用 `exec` 工具创建项目文件夹（如 `项目名_日期`）。
- 按照规划依次编写代码文件，保持代码风格一致。
- 关键函数添加文档字符串（docstring）或注释。
- 如果项目需要多个文件，合理组织目录结构（如 `src/`、`tests/`、`docs/`）。
- 编写依赖文件（如 `requirements.txt`、`package.json`），记录运行环境。

## 4. 自我测试
1. **单元测试**：为每个模块编写测试用例，使用适当的测试框架（pytest、unittest、Jest 等）。
2. **集成测试**：将模块组合运行，验证整体功能是否符合预期。
3. **边界测试**：考虑异常输入、极端情况，确保程序健壮。
4. **性能测试**（如需要）：检查代码运行时间和资源占用。
5. **记录测试结果**：将测试结果保存在项目文件夹内的 `test_report.md` 中。

## 5. 问题处理
- 如果测试失败，定位问题并修复，重复测试直到通过。
- 如果遇到无法解决的技术难题，向 manager 反馈，描述问题和尝试过的解决方案，请求指导或资源。

## 6. 成果归档
1. **整理文件**：将所有代码文件、测试报告、依赖说明、README 放入一个文件夹。
2. **命名文件夹**：格式为 `项目名称_YYYYMMDD`（如 `data_cleaner_20260311`）。
3. **保存**：将整个文件夹复制到 `/home/lcc/.openclaw/共享工作档案/code程序文档/` 下。
4. **报告**：向 manager 发送消息，内容包含：
   - 项目名称和简要功能。
   - 测试结果摘要（通过/失败，覆盖率等）。
   - 绝对路径：`/home/lcc/.openclaw/共享工作档案/code程序文档/项目名称_YYYYMMDD/`

## 7. 响应超时处理
- 任何步骤若超过30秒无响应，清空上下文并重新启动任务。若多次失败，向 manager 报告异常。
